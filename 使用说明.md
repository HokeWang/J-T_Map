## 使用方式

1. 确认电脑已安装 Node.js LTS。没有安装时，到 <https://nodejs.org/> 下载并安装。
2. 双击 `start-app.cmd`。
3. 浏览器会打开 <http://localhost:5174>。
4. 在页面里填写 Google Maps JavaScript API Key。
5. `country` 可以按国家填写两位代码，例如 `CO`。
6. 点击“加载地图”。
7. 导入 CSV 文件，或点击“下载 Excel 导入模板”后按模板填写再导入。
8. 在“地址入参方案”里维护一个或多个字段拼接方案，例如完整地址、详细地址优先等。
9. 点击“配置供应商请求”，维护谷歌、微软、亚马逊等供应商。
10. 在同一个弹窗里配置相距距离阈值和厂商优先级，例如 `Google>Microsoft>Amazon`。
11. 点击“开始多供应商对比”。

## 入参方案

每个入参方案是一组地址字段拼接规则。导入文件后，页面会默认生成两个方案：

* 完整地址
* 方案二：详细地址优先

可以新增、删除、重命名方案，也可以为每个方案勾选不同字段。执行解析时，系统会按“供应商 × 入参方案”逐一请求，例如 `Google / 方案一`、`Google / 方案二`、`Microsoft / 方案一`。

## 供应商配置

每个供应商需要配置：

* 请求 URL，例如 `https://maps.googleapis.com/maps/api/geocode/json`
* 请求参数 Key/Value，写法类似 Postman 的 Params
* 纬度 JSON 路径
* 经度 JSON 路径
* 可选的地址标签 JSON 路径

参数值支持这些占位符：

* `{{address}}`：按页面勾选字段拼接出来的地址
* `{{country}}`：页面输入的国家代码
* `{{googleKey}}`：页面输入的 Google Maps JavaScript API Key

JSON 路径用点号表示，数组下标也用数字。例如 Google 第一条结果的经纬度路径是：

```text
results.0.geometry.location.lat
results.0.geometry.location.lng
```

## 取值规则

“配置供应商请求”弹窗中支持配置相距距离阈值，例如 `100` 米。任意两个“供应商 / 入参方案”结果之间的距离小于或等于阈值时，这两个结果进入候选池。

取值规则：

1. 有候选池时，从候选池中按厂商优先级取最高优先级供应商。
2. 同一供应商有多个入参方案命中时，按页面里的方案顺序取第一个命中的结果。
3. 没有候选池时，直接按厂商优先级取最高优先级供应商，并把置信度标记为“不可信”。
4. 阈值内都视为同等相似，不再比较 20 米和 90 米谁更近。

## Google API 要求

这个工具需要 API Key 启用以下能力：

* Maps JavaScript API
* Geocoding API

页面显示地图会使用 Maps JavaScript API。地址解析会通过本地 Node 服务请求：

```text
https://maps.googleapis.com/maps/api/geocode/json
```

## 文件说明

* `start-app.cmd`：Windows 启动脚本
* `server.js`：本地服务和 Google Geocoding 代理
* `index.html`：页面入口
* `src/`：前端脚本和样式
* `sample-addresses.csv`：示例地址数据

## 注意事项

运行窗口不要关闭。关闭窗口后，本地服务也会停止，页面就无法继续请求地址解析。
